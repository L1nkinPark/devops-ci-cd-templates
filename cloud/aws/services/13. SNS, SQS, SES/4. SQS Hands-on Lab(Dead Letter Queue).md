# 4. SQS Hands-on Lab(Dead Letter Queue)

Bài thực hành này đã được di chuyển sang thư mục thực hành riêng (Labs) của repository để phân tách rõ ràng tài liệu lý thuyết (Services) và hướng dẫn thực hành chi tiết (Labs).

Để xem hướng dẫn thực hiện chi tiết từng bước bao gồm:
1. Tạo một queue phụ làm Dead Letter Queue (DLQ).
2. Cấu hình redrive policy của queue chính với giới hạn Maximum receives.
3. Gửi test message và poll nhiều lần để quan sát Receive Count tăng dần.
4. Kiểm tra message tự động chuyển sang DLQ khi vượt ngưỡng xử lý.
