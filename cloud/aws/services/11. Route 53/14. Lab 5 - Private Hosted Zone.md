# 14. Lab 5 - Private Hosted Zone

## I. Yêu cầu bài thực hành
Bài thực hành hướng dẫn cấu hình và phân giải tên miền nội bộ (Private Hosted Zone) bảo mật trong mạng ảo VPC của AWS.

Các công việc chính cần thực hiện:
1. Khởi tạo **2 EC2 Instance (A/B)** trong cùng một mạng ảo VPC, đảm bảo kích hoạt Public IP và tính năng DNS resolution/DNS hostnames trên VPC.
2. Tạo một **Private Hosted Zone** cho tên miền nội bộ (ví dụ: `test.local`) và liên kết zone này với VPC chứa hai máy chủ trên.
3. Tạo các bản ghi loại **A-Record** (`server-a.test.local` và `server-b.test.local`) trỏ đến địa chỉ Private IP tương ứng của Server A và Server B.
4. Cấu hình **Security Group (SG)**: Mở quy tắc inbound `All ICMP - IPv4` và gán nguồn là chính Security Group đó để các instance có thể giao tiếp ping nội bộ với nhau.
5. Đăng nhập vào Server A (SSH) và thực hiện kiểm tra kết nối bằng lệnh `ping` tới Server B sử dụng tên miền nội bộ `server-b.test.local`.
