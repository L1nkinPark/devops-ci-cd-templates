# 2. CloudFront Key Concepts (Các khái niệm cốt lõi trong CloudFront)

Để thiết lập và vận hành tốt dịch vụ CloudFront, bạn cần hiểu rõ các thành phần và cơ chế hoạt động cốt lõi sau:

---

## I. Các khái niệm cơ bản

### 1. Distribution (Bản phân phối)
AWS CloudFront phân phối nội dung từ nguồn gốc (Origin) đến các điểm biên (Edge Locations) thông qua một **Distribution**. 
* Một Distribution định nghĩa cách CloudFront phân phối nội dung, bao gồm địa chỉ của Origin, các Edge Locations được sử dụng, các thiết lập bảo mật (Security), chứng chỉ SSL/TLS, Caching và Logging.
* Khi tạo xong, Distribution sẽ cấp cho bạn một tên miền mặc định (ví dụ: `d111111abcdef8.cloudfront.net`). Bạn có thể gán tên miền tùy chỉnh (CNAME, ví dụ: `cdn.yourwebsite.com`) thông qua **AWS Certificate Manager (ACM)**.

### 2. Edge Location (Điểm biên)
AWS CloudFront sử dụng một mạng lưới toàn cầu các **Edge Locations** (các máy chủ biên đặt tại các trung tâm dữ liệu trên thế giới).
* Nhiệm vụ chính là phân phối nội dung trực tiếp đến người dùng cuối với độ trễ thấp nhất dựa trên vị trí địa lý của họ.
* Edge Location cũng đóng vai trò giảm tải cho Origin Server và thực hiện các tính năng bảo mật mạng tại biên.

### 3. Origin (Nguồn gốc)
Đây là nơi lưu trữ nội dung gốc (Origin Content) của bạn. Khi xảy ra Cache Miss tại Edge Location, CloudFront sẽ quay về Origin để lấy dữ liệu.
* **Các loại Origin:** Có thể là một Amazon S3 Bucket (lưu trữ tĩnh), một Web Server (ALB, EC2 chạy ứng dụng web động), Amazon API Gateway hoặc bất kỳ Custom HTTP Server nào khác.

### 4. Cache (Bộ nhớ đệm)
AWS CloudFront lưu trữ tạm thời các tài nguyên tại các Edge Locations để giảm thời gian phản hồi và tăng tốc độ tải trang web.
* Các tài nguyên được cache thường bao gồm hình ảnh, video, tệp tin CSS, JavaScript và trang HTML tĩnh.

### 5. Logging and Reporting (Ghi log và Báo cáo)
AWS CloudFront cung cấp các báo cáo trực quan về hoạt động của distribution:
* Ghi lại chi tiết mọi request gửi đến CDN (bao gồm IP, thời gian, kết quả phản hồi HTTP, Edge Location xử lý).
* Cung cấp báo cáo về lưu lượng truyền tải (Data Transfer Out), số lần truy cập (Requests) và tỷ lệ Cache Hit/Miss. Log có thể được cấu hình để tự động lưu về một S3 Bucket.

### 6. Security (Bảo mật tại biên)
AWS CloudFront hỗ trợ nhiều tính năng bảo mật toàn diện:
* Hỗ trợ kết nối mã hóa HTTPS đầu cuối.
* Cho phép đính kèm SSL/TLS Certificate.
* Tích hợp sâu với **AWS Shield** (chống DDoS) và **AWS WAF** (lọc request xấu).

## II. Customize at the Edge (Tùy biến tại biên mạng)

Thông qua cơ chế **Lambda@Edge** và **CloudFront Functions**, AWS cho phép bạn thực thi các đoạn mã (function) trực tiếp tại Edge Locations để xử lý request trước khi gửi về origin hoặc phản hồi lại cho client.

* **Lợi thế:** Xử lý nhanh, giảm độ trễ tối đa và tiết kiệm tải tính toán cho Origin Server.
* **Các trường hợp sử dụng (Use Cases):**
  * **Authentication & Authorization:** Xác thực chữ ký số, kiểm tra quyền truy cập của người dùng ngay tại điểm biên mạng.
  * **SEO / Dynamic Routing:** Định tuyến thông minh dựa trên thiết bị hoặc quốc gia của người dùng.
  * **Realtime Image Transformation:** Tự động tối ưu hoặc thay đổi kích thước ảnh theo thiết bị của client.
  * **A/B Testing:** Điều hướng tỷ lệ phần trăm người dùng sang các phiên bản website khác nhau mà không cần cấu hình ở backend.
  * **Anti-Bot & Security:** Chặn bot rác, kiểm tra header bảo mật.
  * **User Prioritization:** Điều phối hàng đợi cho người dùng khi hệ thống quá tải.

## III. Cơ chế vận hành Cache

### 1. TTL (Time to Live)
TTL quyết định thời gian lưu cache của một tệp tại Edge Location.
* **Min, Max, Default TTL:** Cấu hình thời hạn cache áp dụng khi origin không chỉ định rõ header điều khiển cache.
* **Cache-Control Header:** Cho phép bạn cấu hình thời gian cache trực tiếp từ server gốc (ví dụ: `Cache-Control: max-age=86400`).

### 2. Cache Hit và Cache Miss Ratio
* **Cache Hit:** Request được phục vụ ngay từ bộ nhớ đệm tại Edge Location.
* **Cache Miss:** Request chưa có trong cache hoặc đã quá hạn, CloudFront phải quay về Origin Server lấy dữ liệu mới.
* **Cache Hit Ratio:** Tỷ lệ phần trăm request được phục vụ từ cache. Mục tiêu tối ưu là đẩy con số này lên càng cao càng tốt.

### 3. Invalidations (Xóa Cache chủ động)
Khi cập nhật mã nguồn (ví dụ: upload tệp tin `index.html` mới lên S3), bạn cần yêu cầu CloudFront xóa cache chủ động các tệp tin cũ bằng tính năng **Invalidation** (ví dụ: nhập `/index.html` hoặc `/*` để làm mới).
