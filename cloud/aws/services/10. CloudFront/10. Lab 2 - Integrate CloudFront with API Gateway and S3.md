# 10. Lab 2 – Sử dụng CloudFront kết hợp với API Gateway and S3 (CloudFront Hands-on Lab)

Bài thực hành này đã được chuyển sang thư mục hướng dẫn triển khai (**lab**) của dự án để đảm bảo tính phân tách tài nguyên rõ ràng giữa phần tài liệu giới thiệu dịch vụ (Services) và hướng dẫn thực hành triển khai thực tế (Deploy).

Để xem toàn bộ hướng dẫn thực hành từng bước chi tiết (Hands-on Lab) bao gồm:
1. Tạo một Amazon S3 bucket chứa static assets (Trang chủ tĩnh).
2. Tận dụng API Gateway đã tạo ở các bài Lab trước để xử lý dynamic API requests.
3. Tạo CloudFront Web Distribution với **nhiều Origins** (Multi-Origins): Origin S3 và Origin API Gateway.
4. Cấu hình **Custom Cache Behaviors** để phân tuyến:
   - Các request tĩnh `/` hoặc `/static/*` được định tuyến về S3.
   - Các request động `/dev/*` hoặc `/api/*` được định tuyến về API Gateway.
5. Kiểm thử định tuyến và xác minh hoạt động hệ thống.
