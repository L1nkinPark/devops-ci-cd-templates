# API Gateway Advanced Options

Bên cạnh các tính năng định tuyến cơ bản, Amazon API Gateway cung cấp nhiều tùy chọn nâng cao giúp bạn quản lý, bảo mật, vận hành và phân phối API một cách chuyên nghiệp ở quy mô lớn:

---

## 1. Stage (Môi trường triển khai)
Một **Stage** là một snapshot logic đại diện cho trạng thái cấu hình của API tại một thời điểm triển khai cụ thể. Thông thường, một API sẽ được triển khai lên các Stage tương ứng với các môi trường như `dev`, `staging`, `prod`.

*   **Stage Variables (Biến môi trường):** Định nghĩa các cặp key-value động hoạt động giống như biến môi trường. Bạn có thể sử dụng biến này để trỏ API Gateway đến các backend Lambda hoặc HTTP Server khác nhau tùy thuộc vào Stage hiện tại mà không cần sửa code API (ví dụ: dùng `${stageVariables.lambdaFunction}` làm tên hàm Lambda tích hợp).
*   **Caching (Bộ đệm API):** Hỗ trợ lưu trữ tạm thời các response từ backend dựa trên HTTP method, headers hoặc query parameters. Điều này giúp giảm độ trễ (latency) cho client và giảm thiểu tải/chi phí cho các dịch vụ backend.
*   **Throttling & Rate Limiting:** Thiết lập giới hạn tần suất yêu cầu ở cấp độ stage (ví dụ: tối đa 1000 requests/second) để bảo vệ hệ thống khỏi các cuộc tấn công DDoS hoặc tình trạng quá tải.
*   **Logging & Tracing:** Dễ dàng tích hợp với **Amazon CloudWatch Logs** (để ghi nhận toàn bộ thông tin chi tiết của request/response) và **AWS X-Ray** (để theo dõi, vẽ bản đồ luồng đi của request qua các microservices nhằm debug độ trễ).

---

## 2. Usage Plan (Kế hoạch sử dụng)
**Usage Plan** quy định ai có thể truy cập vào những Stage API nào, với tần suất bao nhiêu và giới hạn dung lượng ra sao trong một khoảng thời gian xác định.

*   **Throttling (Điều tiết):** Thiết lập tốc độ yêu cầu tối đa ổn định (Request Rate - tính bằng requests/second) và lượng requests tăng đột biến tối đa cho phép xử lý ngay lập tức (Burst Limit).
*   **Quota (Hạn mức):** Giới hạn tổng số lượng request tối đa mà người dùng có thể gửi trong một khoảng thời gian dài (ngày, tuần hoặc tháng). Ví dụ: cho phép tối đa 10,000 requests mỗi tháng.
*   **Quản lý đối tác:** Giúp bạn dễ dàng cung cấp các gói dịch vụ API khác nhau (ví dụ: gói Free, gói Basic, gói Premium) cho từng khách hàng hoặc đối tác.

---

## 3. API Key (Khóa API)
**API Key** là các chuỗi ký tự ngẫu nhiên (string tokens) được cấp phát cho khách hàng hoặc đối tác để định danh và đo lường lưu lượng sử dụng.

*   **Cách hoạt động:** Client gửi API Key kèm theo request (thông thường qua HTTP Header `x-api-key`). API Gateway sẽ xác thực key này với Usage Plan đã được liên kết để quyết định cho phép truy cập hay chặn lại.
*   **Metering (Đo lường lưu lượng):** API Key kết hợp với Usage Plan hỗ trợ theo dõi chính xác lượng sử dụng của từng khách hàng phục vụ cho mục đích thống kê hoặc tính phí.
*   **Lưu ý bảo mật:** API Key không nên dùng làm giải pháp bảo mật chính cho API (như bảo vệ thông tin cá nhân). Để xác thực/phân quyền người dùng, hãy ưu tiên sử dụng **Cognito Authorizer** hoặc **Lambda Authorizer**.

---

## 4. Deployment History (Lịch sử triển khai)
Mỗi lần thực hiện Deploy API lên một Stage, API Gateway sẽ ghi nhận lại lịch sử của bản deploy đó dưới dạng các phiên bản (Deployments).

*   **Audit Trail:** Theo dõi ai đã thực hiện thay đổi và deploy vào lúc nào.
*   **Rollback nhanh chóng:** Nếu phiên bản deploy mới nhất phát sinh lỗi ngoài ý muốn trên môi trường Production, bạn có thể dễ dàng chuyển đổi (point) Stage quay trở lại bản deploy hoạt động ổn định trước đó trong lịch sử chỉ với vài click chuột mà không cần sửa đổi hay rollback code thủ công.

---

## 5. Canary Deployment (Triển khai Canary / Thử nghiệm song song)
**Canary Deployment** là kỹ thuật giảm thiểu rủi ro khi cập nhật API bằng cách phân luồng một lượng nhỏ traffic thử nghiệm sang phiên bản API mới (Canary), trong khi phần lớn traffic vẫn đi qua phiên bản cũ ổn định (Production).

*   **Phân luồng Traffic:** Cấu hình tỷ lệ phần trăm phân bổ traffic trực tiếp trên Stage (ví dụ: 10% traffic đi qua API mới, 90% đi qua API cũ).
*   **Giám sát & Đánh giá:** Theo dõi log và chỉ số lỗi (error metrics) của bản Canary trong CloudWatch.
*   **Quyết định hành động:**
    *   **Promote Canary:** Nếu bản mới hoạt động ổn định, nâng cấp bản Canary thành bản chính thức và chuyển 100% traffic qua bản mới.
    *   **Delete Canary:** Nếu phát hiện lỗi hoặc độ trễ cao, lập tức thu hồi toàn bộ traffic về phiên bản cũ an toàn mà không gây ảnh hưởng đến đại đa số người dùng.

## 6. Custom Domain Names (Tên miền tùy chỉnh)
Theo mặc định, API Gateway cấp cho bạn một Endpoint ngẫu nhiên khó nhớ: `https://{api-id}.execute-api.{region}.amazonaws.com/{stage}`. **Custom Domain Names** cho phép bạn ánh xạ API này tới một tên miền chuyên nghiệp của riêng bạn (ví dụ: `api.company.com`).

*   **Bảo mật HTTPS:** Tích hợp với dịch vụ **AWS Certificate Manager (ACM)** để quản lý chứng chỉ SSL/TLS, kích hoạt kết nối mã hóa HTTPS cho tên miền của bạn hoàn toàn miễn phí.
*   **Định tuyến linh hoạt:** Cấu hình DNS (ví dụ: Route 53 hoặc Cloudflare) trỏ bản ghi CNAME hoặc Alias tới Custom Domain Endpoint của API Gateway.
*   **Base Path Mapping (Ánh xạ đường dẫn):** Cho phép bạn gán nhiều API hoặc Stage khác nhau vào cùng một tên miền thông qua tiền tố đường dẫn (Base Paths). Ví dụ:
    *   `api.company.com/v1/users` $\rightarrow$ Trỏ tới `Users-API` Stage `prod`
    *   `api.company.com/v1/orders` $\rightarrow$ Trỏ tới `Orders-API` Stage `prod`
