# 11. Lab 4: ECS Cluster - Khởi tạo Service kết nối Application Load Balancer

**Mục tiêu bài Lab:**
Trong bài lab này, bạn sẽ thực hành cấu hình triển khai thực tế của ứng dụng container trên Amazon ECS dưới dạng một Service thay vì chạy Task đơn lẻ. Bạn sẽ cấu hình kết hợp một Application Load Balancer (ALB) và một Target Group ở chế độ IP addresses để cân bằng tải cổng 80 cho 2 Task Fargate chạy song song, đồng thời tắt IP công cộng (Public IP) của Task để bảo mật.

## Tóm tắt các cấu phần chính trong bài Lab:

1. **Khởi tạo Target Group (Type: IP):**
   - Định nghĩa Target Group `test-ecs-tg` cổng 80 HTTP với Target type chọn `IP addresses`. Để trống danh sách IP đăng ký ban đầu để ECS Service tự quản lý.

2. **Cấu hình Application Load Balancer (ALB):**
   - Khởi tạo Load Balancer `test-ecs-alb` ở chế độ Internet-facing, phân phối lưu lượng vào các Subnets khác nhau để đảm bảo High Availability, cấu hình Listener HTTP chuyển hướng tới `test-ecs-tg`.

3. **Tạo ECS Service:**
   - Triển khai Service `test-ecs-service` trên cụm `test-cluster-ecs` với số lượng mong muốn **Desired tasks = 2**.
   - Cấu hình tắt **Public IP (Disabled)** của Task, liên kết container `my-httpd` cổng 80 với ALB và Target Group tương ứng.

4. **Kiểm chứng khả năng tự phục hồi (Self-Healing):**
   - Truy cập trang web CMC qua DNS của ALB. Thực hiện stop thử 1 Task để kiểm tra tính năng tự tạo mới Task thay thế của ECS Service để duy trì đúng cấu hình mong muốn.
