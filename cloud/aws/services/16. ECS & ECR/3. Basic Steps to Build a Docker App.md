# 3. Các bước cơ bản để xây dựng ứng dụng Docker

Để xây dựng và chạy một ứng dụng bằng Docker, bạn thường sẽ trải qua các bước cơ bản sau:

1. **Chuẩn bị Dockerfile:** 
   `Dockerfile` là một tệp văn bản (đóng vai trò như công thức) mô tả các bước cần thiết để tạo ra môi trường container chứa ứng dụng của bạn. Nó bao gồm khai báo hệ điều hành gốc (base image), các lệnh cài đặt thư viện/công cụ hệ thống, sao chép mã nguồn và thiết lập runtime (thời gian chạy).

2. **Xây dựng Docker Image:** 
   Sử dụng lệnh `docker build` để xây dựng Docker Image từ `Dockerfile`.
   *(Ví dụ: `docker build -t my-app:latest .`)*

3. **Kiểm tra Docker Image:** 
   Sử dụng lệnh `docker images` để kiểm tra danh sách các ảnh Docker (images) hiện có sẵn trên máy tính của bạn. Hãy đảm bảo rằng Docker Image của ứng dụng của bạn đã được xây dựng thành công và xuất hiện trong danh sách này.

4. **Chạy Docker Container:** 
   Sử dụng lệnh `docker run` để khởi tạo và chạy một container thực tế từ Docker Image vừa tạo. 
   *(Ví dụ: `docker run -d -p 8080:80 my-app:latest`)*

5. **Kiểm tra ứng dụng:** 
   Truy cập ứng dụng của bạn thông qua địa chỉ IP (thường là `localhost` đối với máy cá nhân) hoặc tên miền cùng với port đã chỉ định (ví dụ port `8080`).

> [!NOTE]
> **Lưu ý:** Quy trình 5 bước trên là kịch bản chuẩn khi chạy trên máy local. Trong thực tế, khi triển khai ứng dụng lên môi trường Cloud, quy trình sẽ bao gồm nhiều bước phức tạp hơn như: Push image lên Container Registry (ECR), cấu hình các orchestrator như ECS hoặc EKS, thiết lập Load Balancer, và tích hợp các công cụ CI/CD (Pipeline).
