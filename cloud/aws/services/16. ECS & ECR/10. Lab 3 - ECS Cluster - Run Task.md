# 10. Lab 3: ECS Cluster - Khởi chạy Task trên Fargate

**Mục tiêu bài Lab:**
Trong bài lab này, bạn sẽ thực hành khởi tạo một ECS Cluster ở chế độ Fargate (Serverless). Sau đó, bạn sẽ tạo một Task Definition cấu hình CPU/RAM tối thiểu, trỏ đến Docker image từ Amazon ECR ở bài Lab trước, và thực hiện khởi chạy một Task độc lập (standalone task) với cấu hình Public IP và Security Group để kiểm tra truy cập ứng dụng web chạy trực tiếp trên AWS.

## Tóm tắt các cấu phần chính trong bài Lab:

1. **Khởi tạo Cluster Fargate:**
   - ECS Cluster quản lý các nhóm tài nguyên chạy ứng dụng. Đối với Fargate, hạ tầng máy chủ được AWS tự động cấp phát và quản trị hoàn toàn.

2. **Cấu hình Task Definition:**
   - Định nghĩa family `my-httpd-task-definition`, launch type compatibility `FARGATE`, gán dung lượng `0.25 vCPU` và `0.5 GB RAM` cùng image URI của repository ECR `my-httpd`.

3. **Chạy Standalone Task:**
   - Thực thi chạy Task Fargate. Kích hoạt cờ **Public IP (Enabled)** và liên kết với một Security Group cho phép lưu lượng truy cập Inbound cổng **80 (HTTP)** từ internet.

4. **Kiểm nghiệm truy cập:**
   - Sử dụng địa chỉ Public IP tạm thời của Task đang ở trạng thái `RUNNING` để kiểm tra kết quả hiển thị trên trình duyệt web.
