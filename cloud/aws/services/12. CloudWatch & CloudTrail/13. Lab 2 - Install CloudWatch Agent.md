# 13. Lab 2 - Install CloudWatch Agent

## I. Yêu cầu bài thực hành
Bài thực hành hướng dẫn cấu hình và cài đặt CloudWatch Agent trên máy chủ EC2 Linux nhằm thu thập các tệp tin logs hệ thống, logs lỗi/truy cập của ứng dụng Apache Web Server, và các chỉ số đo lường tài nguyên tùy chỉnh (Memory, Disk Space) mà CloudWatch không thể tự động thu thập từ phía ngoài hypervisor.

Các công việc chính cần thực hiện:
1. Tạo một **IAM Role** cho phép EC2 ghi dữ liệu logs/metrics lên CloudWatch (`CloudWatchAgentServerPolicy`), sau đó gắn Role này vào instance EC2.
2. Cài đặt gói phần mềm **Amazon CloudWatch Agent** bằng trình quản lý gói `dnf` trên Amazon Linux 2023.
3. Chạy trình hướng dẫn cấu hình tương tác **CloudWatch Agent Config Wizard** để sinh cấu hình `/opt/aws/amazon-cloudwatch-agent/bin/config.json` thu thập:
   * **Host Metrics**: RAM (`mem_used_percent`), ổ đĩa (`used_percent`) với chu kỳ thu thập 30 giây (High-resolution).
   * **Log Files**: Thu thập `/var/log/httpd/access_log` (Log Group `test-server/access-log`) và `/var/log/httpd/error_log` (Log Group `test-server/error_logs`) với cấu hình retention (thời gian lưu trữ) là 90 ngày.
4. Kích hoạt và khởi chạy CloudWatch Agent service trên EC2 Linux.
5. Truy cập CloudWatch Console để xác thực Logs đã đẩy lên các Log Groups và custom metrics hiển thị trong Namespace `CWAgent`.
