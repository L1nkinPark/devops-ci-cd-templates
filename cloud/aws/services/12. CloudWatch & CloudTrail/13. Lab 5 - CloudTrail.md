# 13. Lab 5 - CloudTrail

## I. Yêu cầu bài thực hành
Bài thực hành hướng dẫn cấu hình tạo một Trail mới trong AWS CloudTrail để lưu trữ vĩnh viễn lịch sử cuộc gọi API trên tài khoản sang một S3 bucket an toàn phục vụ kiểm toán bảo mật, đồng thời thực hiện tra cứu truy vết hành động xóa tài nguyên thực tế.

Các công việc chính cần thực hiện:
1. Khởi tạo một S3 Bucket mới phục vụ làm nơi lưu trữ logs.
2. Tạo một **Trail** mới trong AWS CloudTrail, chọn áp dụng cho tất cả các Regions, và cấu hình lưu trữ log nén tự động về S3 Bucket vừa tạo.
3. Thực hiện một hành động thay đổi hạ tầng thực tế (ví dụ: tạo và xóa một EC2 Security Group hoặc một S3 Bucket phụ).
4. Truy cập giao diện **CloudTrail Event History** để tìm kiếm cuộc gọi API tương ứng (`CreateSecurityGroup` hoặc `DeleteSecurityGroup`), truy vết chính xác thông tin: Ai đã xóa, lúc mấy giờ, từ địa chỉ IP nào.
5. Kiểm tra tệp tin log thô định dạng JSON được CloudTrail tự động lưu trữ trong S3 bucket.
