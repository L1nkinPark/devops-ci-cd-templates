# 1. Amazon VPC Hands-on Lab (Thiết kế VPC và các thành phần)

Bài thực hành này hướng dẫn từng bước thiết kế và xây dựng một kiến trúc mạng ảo Amazon VPC tùy chỉnh (Custom VPC) với khả năng cô lập tài nguyên bảo mật cao trên môi trường AWS Console. 

Mô hình thiết kế sẽ áp dụng chia các phân khu mạng con (Subnets) có kích thước `/22` từ dải mạng chính `/16`, dự phòng đa vùng khả dụng (Multi-AZ) và cấu hình điều phối các luồng traffic qua Internet Gateway, NAT Gateway và Route Tables.

---

## I. Kịch bản Thiết kế Hệ thống Mạng (Network Design Specification)

Chúng ta sẽ xây dựng hạ tầng mạng ảo đáp ứng các thông số kỹ thuật sau:

### 1. Dải IP mạng chính (VPC CIDR Block)
*   **VPC CIDR:** `10.0.0.0/16` (Tổng cộng $65,536$ địa chỉ IP).

### 2. Availability Zones (Phân khu khả dụng)
*   **AZ 1:** `us-east-1a`
*   **AZ 2:** `us-east-1b`

### 3. Phân hoạch Subnets (Mạng con)
Mỗi subnet sử dụng block CIDR **`/22`** (chứa $1,024$ địa chỉ IP, trong đó có $1,019$ IP khả dụng sau khi trừ đi 5 IP hệ thống của AWS):

| Tên Subnet | Availability Zone | Dải IP CIDR | Loại Subnet | Chức năng dự kiến |
| :--- | :--- | :--- | :--- | :--- |
| **Public-Subnet-1A** | `us-east-1a` | `10.0.0.0/22` | Public | Đặt Web Servers, ALB, NAT Gateway |
| **Public-Subnet-1B** | `us-east-1b` | `10.0.4.0/22` | Public | Đặt Web Servers, ALB dự phòng |
| **Private-Subnet-1A**| `us-east-1a` | `10.0.8.0/22` | Private (with NAT) | Đặt App Server, Application Containers |
| **Private-Subnet-1B**| `us-east-1b` | `10.0.12.0/22` | Private (with NAT) | Đặt App Server, Containers dự phòng |
| **Trusted-Subnet-1A**| `us-east-1a` | `10.0.16.0/22` | Isolated Private | Đặt Database (RDS MySQL, PostgreSQL) |
| **Trusted-Subnet-1B**| `us-east-1b` | `10.0.20.0/22` | Isolated Private | Đặt Database dự phòng (Standby Replica) |
| **Mgmt-Subnet-1A**   | `us-east-1a` | `10.0.24.0/22` | Private/Isolated | Đặt máy chủ quản trị (Bastion Host, Jump Box) |
| **Mgmt-Subnet-1B**   | `us-east-1b` | `10.0.28.0/22` | Private/Isolated | Dự phòng máy chủ quản trị |

## II. Các bước thực hiện chi tiết (Step-by-Step Implementation)

### Bước 1: Khởi tạo Custom VPC
1. Đăng nhập vào **AWS Management Console** và điều hướng tới dịch vụ **VPC**.
2. Tại menu bên trái, chọn **Your VPCs**, sau đó nhấn nút **Create VPC**.
3. Cấu hình các thông số sau:
   *   **Resources to create:** Chọn `VPC only` để tự cấu hình thủ công từng bước (giúp hiểu rõ bản chất).
   *   **Name tag:** `Main-VPC`
   *   **IPv4 CIDR block:** Chọn `IPv4 CIDR manual input`.
   *   **IPv4 CIDR:** Điền `10.0.0.0/16`.
   *   **Tenancy:** Chọn `Default`.
4. Nhấn **Create VPC**.

### Bước 2: Tạo các Subnets tương ứng với thiết kế
1. Tại menu bên trái, chọn **Subnets**, sau đó nhấn nút **Create subnet**.
2. Chọn VPC đích là **Main-VPC** vừa tạo.
3. Tiến hành tạo lần lượt 8 subnet theo bảng thông số thiết kế ở phần I. Để tạo nhanh, bạn có thể nhấn **Add new subnet** để cấu hình nhiều subnet cùng lúc:
   *   *Ví dụ cấu hình Subnet 1:*
       *   **Subnet name:** `Public-Subnet-1A`
       *   **Availability Zone:** `us-east-1a`
       *   **IPv4 CIDR block:** `10.0.0.0/22`
   *   *Ví dụ cấu hình Subnet 3:*
       *   **Subnet name:** `Private-Subnet-1A`
       *   **Availability Zone:** `us-east-1a`
       *   **IPv4 CIDR block:** `10.0.8.0/22`
4. Sau khi nhập đủ thông tin cho các Subnets, nhấn **Create subnet**.

### Bước 3: Tạo và Gắn Internet Gateway (IGW)
Internet Gateway giúp kết nối các tài nguyên trong Public Subnet ra ngoài Internet.
1. Chọn **Internet Gateways** tại menu bên trái, nhấn **Create internet gateway**.
2. Đặt tên **Name tag:** `Main-IGW`, nhấn **Create internet gateway**.
3. Hệ thống hiển thị trạng thái `Detached`. Chọn nút **Actions** ở góc trên bên phải → Chọn **Attach to VPC**.
4. Chọn VPC **Main-VPC** từ danh sách thả xuống, nhấn **Attach internet gateway**.

### Bước 4: Khởi tạo NAT Gateway cho phân khu Private
NAT Gateway giúp các tài nguyên trong Private Subnet có thể kết nối ra ngoài internet (ví dụ để tải bản cập nhật hệ điều hành, cài thư viện) nhưng ngăn chặn internet truy cập trực tiếp vào chiều ngược lại.
1. Chọn **NAT Gateways** tại menu bên trái, nhấn **Create NAT gateway**.
2. Cấu hình các thông số:
   *   **Name:** `Main-NAT-GW-1A`
   *   **Subnet:** Chọn **Public-Subnet-1A** (Bắt buộc phải đặt NAT Gateway trong Public Subnet để nó có thể định tuyến ra ngoài qua Internet Gateway).
   *   **Connectivity type:** Chọn `Public`.
   *   **Elastic IP allocation ID:** Nhấn nút **Allocate Elastic IP** để AWS tự động cấp phát một IP tĩnh công cộng cho NAT Gateway.
3. Nhấn **Create NAT gateway**. 
*(Lưu ý: Quá trình khởi tạo NAT Gateway có thể mất từ 2-3 phút để chuyển sang trạng thái `Available`).*

### Bước 5: Cấu hình các Route Table (Bảng định tuyến)
Để điều phối traffic theo đúng vai trò bảo mật của từng phân khu, ta sẽ tạo các Route Table riêng biệt:

#### 5.1. Cấu hình Public Route Table (Định tuyến cho Public Subnets)
1. Chọn **Route tables** ở menu bên trái, nhấn **Create route table**.
2. Đặt tên **Name tag:** `Public-RT`, chọn VPC là **Main-VPC**, nhấn **Create route table**.
3. **Thêm Route ra Internet:** Chọn tab **Routes** phía dưới → Nhấn **Edit routes** → Nhấn **Add route**:
   *   **Destination:** `0.0.0.0/0` (Toàn bộ traffic đi ra ngoài internet).
   *   **Target:** Chọn `Internet Gateway` → Chọn **Main-IGW**.
   *   Nhấn **Save changes**.
4. **Liên kết Subnet:** Chọn tab **Subnet associations** → Nhấn **Edit subnet associations**:
   *   Tích chọn **Public-Subnet-1A** và **Public-Subnet-1B**.
   *   Nhấn **Save associations**.

#### 5.2. Cấu hình Private Route Table (Định tuyến cho Private Subnets)
1. Tạo một Route Table mới đặt tên là `Private-RT`, chọn VPC **Main-VPC**, nhấn **Create**.
2. **Thêm Route ra Internet qua NAT:** Chọn tab **Routes** → Nhấn **Edit routes** → Nhấn **Add route**:
   *   **Destination:** `0.0.0.0/0`
   *   **Target:** Chọn `NAT Gateway` → Chọn **Main-NAT-GW-1A**.
   *   Nhấn **Save changes**.
3. **Liên kết Subnet:** Chọn tab **Subnet associations** → Nhấn **Edit subnet associations**:
   *   Tích chọn **Private-Subnet-1A** và **Private-Subnet-1B**.
   *   Nhấn **Save associations**.

#### 5.3. Cấu hình Isolated/Trusted Route Table (Định tuyến cho Database Subnets)
Phân khu Database cần bảo mật tuyệt đối, không có route ra ngoài internet (ngay cả qua NAT).
1. Tạo một Route Table mới đặt tên là `Isolated-RT`, chọn VPC **Main-VPC**, nhấn **Create**.
2. Giữ nguyên mặc định (chỉ có duy nhất route local `10.0.0.0/16` -> `local` để liên kết nội bộ).
3. **Liên kết Subnet:** Chọn tab **Subnet associations** → Nhấn **Edit subnet associations**:
   *   Tích chọn **Trusted-Subnet-1A** và **Trusted-Subnet-1B**.
   *   Nhấn **Save associations**.

## III. Kiểm tra và Xác minh (Validation)

Để xác thực hệ thống mạng hoạt động đúng thiết kế:

1.  **Kiểm tra tính kết nối của Public Subnet:**
    *   Khởi tạo một EC2 Instance chạy hệ điều hành Linux trong `Public-Subnet-1A`, bật tùy chọn tự động cấp Public IP.
    *   Sử dụng SSH kết nối vào EC2 này từ máy local của bạn. Kết nối thành công chứng minh Internet Gateway và Public Route Table hoạt động tốt.
    *   Từ instance này, chạy lệnh `ping google.com` để xác nhận máy chủ có kết nối internet.
2.  **Kiểm tra tính kết nối và bảo mật của Private Subnet:**
    *   Khởi tạo một EC2 Instance thứ hai trong `Private-Subnet-1A` (không cấp Public IP).
    *   Từ máy Public EC2 ở bước 1, thực hiện SSH vào máy Private EC2 này (Sử dụng kỹ thuật SSH Agent Forwarding hoặc copy private key).
    *   Từ máy Private EC2, chạy lệnh `ping google.com` hoặc `curl https://www.google.com`. Việc truy cập thành công chứng minh NAT Gateway đang chuyển tiếp traffic ra internet tốt.
    *   Từ máy tính cá nhân ở nhà của bạn, thử kết nối SSH trực tiếp tới IP Private của máy Private EC2. Kết nối thất bại chứng minh phân khu Private được cô lập an toàn khỏi sự tiếp cận trực tiếp từ môi trường internet công cộng.
