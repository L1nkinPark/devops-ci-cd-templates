# 5. Lab 1: Build và chạy custom image sử dụng Dockerfile

Bài thực hành này hướng dẫn bạn cách viết một `Dockerfile` cơ bản để đóng gói ứng dụng web tĩnh chạy trên nền Web Server Apache (`httpd`). Sau đó, bạn sẽ thực hiện build Docker Image và khởi chạy Docker Container trên máy tính cá nhân để kiểm tra kết quả.

---

## I. Mục tiêu bài Lab
* Hiểu cách cấu trúc và viết một `Dockerfile`.
* Biết cách sử dụng lệnh `docker build` để đóng gói ứng dụng thành một Docker Image.
* Biết cách sử dụng lệnh `docker run` để khởi chạy và map cổng (port mapping) từ máy host vào Container.
* Làm quen với các lệnh quản lý container cơ bản (`docker ps`, `docker logs`, `docker stop`, `docker rm`).

### Bước 2: Khởi tạo và viết Dockerfile
`Dockerfile` là tệp chứa các chỉ thị (instructions) để Docker Engine tự động hóa việc build image. Tạo tệp tên là `Dockerfile` (không có đuôi mở rộng) nằm cùng cấp với file `index.html` với nội dung như sau:

* **File:** [Dockerfile](Dockerfile)

```dockerfile
# Sử dụng base image Apache HTTP Server chính thức từ Docker Hub
FROM httpd

# Sao chép file index.html từ máy host vào thư mục phục vụ web mặc định của Apache
COPY index.html /usr/local/apache2/htdocs/

# Thay đổi quyền sở hữu file index.html cho user www-data chạy server Apache
RUN chown www-data:www-data /usr/local/apache2/htdocs/index.html

# Chỉ định lệnh chạy mặc định để chạy Apache ở chế độ foreground
CMD ["httpd-foreground"]
```

> [!NOTE]
> **Giải thích các chỉ thị trong Dockerfile:**
> * **`FROM httpd`**: Định nghĩa base image là Apache HTTP Server bản mới nhất (latest). Đây là môi trường gốc đã được cài đặt sẵn web server.
> * **`COPY index.html /usr/local/apache2/htdocs/`**: Sao chép tệp `index.html` từ máy host vào thư mục webroot mặc định của Apache bên trong container.
> * **`RUN chown www-data:www-data /usr/local/apache2/htdocs/index.html`**: Chạy lệnh thay đổi quyền sở hữu (owner) của tệp tin vừa copy sang user/group `www-data` để đảm bảo Apache Server có quyền đọc ghi và phục vụ tệp mà không bị lỗi Permission Denied.
> * **`CMD ["httpd-foreground"]`**: Thiết lập câu lệnh mặc định sẽ chạy ngay khi container được khởi tạo, nhằm giữ cho tiến trình web server Apache chạy liên tục ở foreground (nếu tiến trình này dừng, container sẽ dừng theo).

### Bước 3: Xây dựng Docker Image (Build Image)
Mở terminal (PowerShell hoặc Command Prompt trên Windows) và di chuyển vào thư mục chứa `Dockerfile` và `index.html` của bài Lab này. Thực thi lệnh sau để bắt đầu build Docker Image:

```bash
docker build -t my-httpd .
```

> [!IMPORTANT]
> **Giải thích cú pháp lệnh build:**
> * **`docker build`**: Lệnh yêu cầu Docker đóng gói image.
> * **`-t my-httpd`**: Tham số đặt tên (tag) cho image là `my-httpd`. Nếu không ghi rõ tag version thì Docker mặc định gán tag là `latest`.
> * **`.` (dấu chấm ở cuối)**: Chỉ định thư mục hiện tại làm **Build Context** cho Docker tìm kiếm Dockerfile và sao chép các tệp tin liên quan.

*Docker sẽ tiến hành tải image `httpd` gốc từ Docker Hub về, tạo layer mới chứa file `index.html`, thực thi đổi owner và đóng gói thành image hoàn chỉnh.*

### Bước 5: Khởi chạy Docker Container (Run Container)
Bây giờ, chúng ta sẽ chạy container từ Docker Image `my-httpd` vừa build bằng lệnh:

```bash
docker run -dit --name my-httpd-app -p 8080:80 my-httpd
```

> [!TIP]
> **Giải thích chi tiết các tham số:**
> * **`-dit`**: Sự kết hợp của ba cờ:
>   - `-d` (Detached): Chạy container chạy ngầm dưới background để không chặn terminal hiện tại.
>   - `-i` (Interactive): Giữ kết nối stdin mở để tương tác với container khi cần.
>   - `-t` (TTY): Phân bổ một terminal ảo (pseudo-TTY) hỗ trợ tương tác dòng lệnh.
> * **`-p 8080:80` (Port mapping)**: Ánh xạ cổng. Mọi truy cập vào cổng **8080 của máy host** sẽ được forward vào cổng **80 (HTTP) bên trong container**.
> * **`--name my-httpd-app`**: Đặt tên gợi nhớ cho container đang chạy là `my-httpd-app`.
> * **`my-httpd`**: Tên của Docker Image dùng để dựng container.

### Bước 7: Các lệnh quản lý và dọn dẹp Container cơ bản

Khi làm việc với Docker, hãy sử dụng các lệnh sau để quản lý tài nguyên của bạn:

1. **Xem các container đang chạy:**
   ```bash
   docker ps
   ```
2. **Xem nhật ký hoạt động (logs) của Apache:**
   ```bash
   docker logs my-httpd-app
   ```
3. **Dừng container đang chạy:**
   ```bash
   docker stop my-httpd-app
   ```
4. **Khởi động lại container đã bị dừng trước đó:**
   ```bash
   docker start my-httpd-app
   ```
5. **Xóa bỏ container (container cần được stop trước khi xóa):**
   ```bash
   docker rm my-httpd-app
   ```
   *(Hoặc xóa cưỡng chế container đang chạy bằng: `docker rm -f my-httpd-app`)*
6. **Xóa Docker Image cục bộ:**
   ```bash
   docker rmi my-httpd
   ```
