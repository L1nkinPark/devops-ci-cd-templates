## II. Tổng quan bài Lab (Yêu cầu)

Bài thực hành này hướng dẫn cấu hình và kiểm chứng cơ chế bảo mật cho API Gateway bằng cách kết hợp với Amazon Cognito User Pool thông qua Cognito Authorizer:

1. **Khởi tạo Cognito Authorizer trên API Gateway:**
   * Tạo mới một Authorizer mang tên `cognito-auth`.
   * Liên kết với Cognito User Pool `test-user-pool-01` đã tạo ở các bài Lab trước.
   * Cấu hình Token source là `method.request.header.authorization` (Client đính kèm token trong Header `authorization`).

2. **Cấu hình Method Request:**
   * Cấu hình phương thức `POST` của resource `/calculate` sử dụng Authorizer `cognito-auth` thay vì `NONE`.
   * Giữ nguyên yêu cầu API Key nếu có cấu hình từ trước để kết hợp đa lớp bảo mật.

3. **Deploy API Gateway:**
   * Thực hiện Deploy API lên Stage `dev` để áp dụng các thay đổi cấu hình mới.

4. **Kiểm thử và xác minh hoạt động:**
   * Gửi request không có Header `authorization` hoặc token sai -> Nhận lỗi `401 Unauthorized`.
   * Đính kèm JWT ID Token lấy được từ Lab 4 vào Header `authorization` -> Gọi API thành công và nhận kết quả từ Lambda backend.
