## II. Tổng quan bài Lab (Yêu cầu)

Bài Lab này hướng dẫn bạn cách bảo mật và quản lý tần suất gọi API (Rate Limiting/Throttling) cho REST API đã tạo ở Lab 1 bằng cách tích hợp **API Key** và **Usage Plan**:

1. **Cấu hình trên API Gateway:**
   * Khởi tạo một **Usage Plan** quy định tốc độ điều tiết (Throttling - Rate/Burst) và hạn mức cuộc gọi (Quota - Số request/tháng).
   * Khởi tạo một **API Key** tự động trên API Gateway.
   * Gán API Key vừa tạo vào Usage Plan.
   * Gán Stage `dev` của API `test-api` vào Usage Plan.
   * Bật tùy chọn **API key required** cho method `POST /calculate` của API `test-api`.
   * Triển khai lại (Redeploy) API Gateway lên Stage `dev` để các cấu hình có hiệu lực.

2. **Kiểm thử trên Postman:**
   * Gửi request không kèm API Key $\rightarrow$ Kiểm tra lỗi `403 Forbidden` trả về từ API Gateway.
   * Bổ sung header `x-api-key` chứa chuỗi khóa bí mật vào request $\rightarrow$ Kiểm tra kết quả phản hồi thành công từ backend Lambda.
